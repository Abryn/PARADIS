Question1: Which of the two programs, Program1 and Program2, has best performance, or are the
performance of the two programs about the same?

Program2 has significantly better performance (~1s) vs (~8s) for my computer. Program2 distributes the workload
across all available cores in the cpu by processing webPages concurrently within each stage while program1 uses
the producer-consumer pattern where the parallelism is limited by the number of stages and synchronizes using
blockingQueues, which increases waiting time between stages.

Question2: Is there any difference in which order different work tasks are performed in the two
different solutions, Program1 and Program2?

Yes. In program1 the execution follows a pipeline pattern where downloading, analyzing and categorizing can happen
simultaneously for different webPages. As soon as a webPage has been downloaded it can be analyzed, and as soon as it
has been analyzed it can be categorized. This means that different webPages can exist in different stages and overlap,
but only there will only be 3 concurrent stages working.

In program2 all downloads will be done in parallel before a single webPage is analyzed. This is different from program1
where webPages are at all three stages simultaneously. Once all webPages are downloaded will the analyzing begin in
parallel, and then after that the categorizing. The stages don't overlap like they do in program1.
